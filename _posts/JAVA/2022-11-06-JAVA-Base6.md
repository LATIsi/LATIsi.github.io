# Generic

Generic은 `실행하기전에 어떤 객체가 담겼는지 알수 없고, 엉뚱하게 사용되는 문제를 가지고 있는` 오브젝트 타입의 컬렉션과는 다르게 `좀 더 강하게 검사하고 타입변환을 제거하는게 큰 특징`이다.

<br>

즉, 클래스나 메서드 내부에서 사용할 내부데이터들을 외부에서 사용자의 필요로 의해 특정 타입으로 지정해 컴파일시에 일반화하는것. Generic의 장점은 아래와 같다.

<br>

- 컴파일에서 강한 타입체크를 하여 오류를 미리 잡을수 있다.
- 타입변환(형변환)을 제거해준다. `오버헤드를 줄여준다!`
- 코드의 재사용성이 높다!

<br>

비 제너릭은 불필요한 타입변환을 하기 때문에 프로그램 성능에 악영향을 끼친다.  
또한 매번 확인하기도 힘들며 같은 배열에서 꺼내는게 다르면 형변환을 일일히 해줘야하기에 불편하므로 Generic을 사용하는게 편하다.

<br>

<br>

## Generic을 사용하는 방법.

`객체 <데이터타입> 객체명 = new 객체<데이터타입> () ;`

<br>

### get

unBoxing개념. 값을 꺼낼때 사용된다

<br>

### set

Boxing개념. 값을 재설정할때 사용한다.

<br>

<br>

매개변수가 하나만 있다면 각각 타입을 담을때 Generic으로 각자 정수, 문자형으로 get과 set을 사용해 값을 담을수가있다.

<br>

## 멀티타입의 패러미터

매개변수가 여러개 있을시 사용.  
Generic 메서드는 `매개타입, 리턴타입`으로 타입패러미터를 갖는것을 말한다.

<br>

선언방법으로

- 리턴 타입 앞에 부등호기호를 추가하고 해당 안에 타입매개변수를 기술한다.
- 리턴타입하고 매개타입으로 타입 매개변수를 사용한다.

가 있다.

<br>

예시들로

`public static <T> T getLast (T[]k) { return k[k.length-1];} `  
리턴타입을 미리 `<T>` 를 하듯 명시해야한다.

<br>

`classname.<타입명> 메서드이름();`  
가시적으로 명확하게 메서드의 타입이 무엇인지 알려준다.

# collection Framewark

콜렉션에 사용되는 인터페이스는

<br>

- 리스트 (List) `ArrayList, Vector, Linked List ...`
- 셋 (Set) `Hashset, TreeSet ....`
- 큐 (Queue) `Deque, Linked List ... `

<br>

등이 있다.

<br>

각자의 특징으로는, 아래의 표와 같다.

<br>

| collection |                                    특징                                    |
| ---------- | :------------------------------------------------------------------------: |
| `List`     |                    객체의 순서가 `있고`, 원소 중복 가능                    |
| `Set`      | 객체의 순서가 `없고`, 동일한 원소여야만 하며 중복은 안된다. 집합과 가깝다. |
| `queue`    |               객체를 `입력한 순서대로 저장`, 원소 중복 가능                |

<br>

## List collection

| List 예시     |                                                                                                                                        설명                                                                                                                                        |
| ------------- | :--------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------: |
| `Array List`  |                                                                 사이즈 고정시 에러가 발생하는 배열과는 다르게 <br> Array List는 필요한 용량만큼 추가할당을 해준다. 읽거나 검색하는것에 유리하며, 동기화가 안된다.                                                                  |
| `Vector`      | 동기화된 메서드로 구성되어 있어서, 멀티 스레드가 동시에 메서드를 실행할수가 없다.<br>즉, 하나의 스레드가 실행을 완료해야지만 다른 스레드를 사용할수가 있다.<br>멀티 스레드 환경에서 객체를 안전하게 추가/삭제할수 있으므로, `중복삭제나 끼어드는 오류가 없어` 스레드가 안정적이다. |
| `Linked List` |                                                                                                    삽입삭제에 유리한 List이다. Array List보다 입력,삭제하는것이 유리한 리스트.                                                                                                     |

<br>

### Array List 관련 추가 설명

ArrayList에 사용되는 메서드들이 있다.
`List<String> a = new ArrayList<String>();` 으로 선언을 먼저 해주어야한다.

<Br>

| ArrayList 메서드        |                                       설명                                        |
| ----------------------- | :-------------------------------------------------------------------------------: |
| `a.add("추가할 내용");` |                           ArrayList에 값을 집어넣는다.                            |
| `a.size( );`            |                    안에 들어간 데이터의 수(사이즈)를 알려준다.                    |
| `a.get(n);`             |                            n번째 순번의 값을 보여준다.                            |
| `a.remove(n);`          | n번째 순번의 데이터를 삭제한다. 순번을 써도 괜찮지만, 원소이름을 넣어도 삭제가능. |

<br>

또한 ArrayList는 필요한 용량만큼 추가 할당이지만, 고정된 ArrayList도 선언 가능하다.

<br>

`List<String> a = Arrays.asList("값1","값2","값3", ....);`

<br>

값을 꺼낼시, for in을 쓰면 편하다.

## set collection

`집합 컬렉션`이라고도 하며, 저장순서가 없다.  
객체를 중복 저장할 수 없고 Null같은것도 딱 하나만 가능하다.

<br>

| set collection |                                                                                                              설명                                                                                                               |
| -------------- | :-----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------: |
| `HashSet<>`    | 객체를 조정하기전에 먼저 `객체 해시코드 메서드로 해시코드를 알아낸다.`<br>이미 저장되어 있는 객체들의 해시태그랑 비교하고 `동일한 해시태그가 있을시 = 으로 리턴값을 비교, true가 나올시 동일객체로 판단해 중복저장을 막아준다.` |
| `LinkedSet`    |                                                                                               데이터를 입력한 순서대로 저장한다.                                                                                                |
| `TreeSet`      | `데이터를 tree형태로 저장하는` 집합인 set을 기반으로 한 Tree 구조의 컬렉션. 데이터가 항상 정렬되어있다. <br>좌우에 자식노드를 참조하기위한 변수들로 구성이 되어있으며 하나의 노드에 `왼쪽변수, 노드의 값, 오른쪽 변수`로 구성.  |

<br>

위 세개에서 공통적으로 쓰이는 추가, 검색, 삭제메서드가 있다.

<br>

| 메서드 이름                     |                                                                                                                                                                                     설명                                                                                                                                                                                     |
| ------------------------------- | :--------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------: |
| `add()`                         |                                                                                                                 객체를 가져와 집어넣을때 사용하는 메서드. return값은 성공시 true, 실패시 false이다.<br>매개변수로는 객체가 들어가며 제네릭에 영향을 받는다.                                                                                                                  |
| `clear(), remove()`             |                                                                                                   객체 삭제시 쓰이는 메서드들.<br> clear()는 리턴과 매개변수도 없지만 remove()는 매개변수로 객체를 주면 그 객체를 찾아 삭제하고 삭제 완료시 true, 아닐시 false를 반환한다.                                                                                                   |
| `isEmpty(), contains(), size()` |                                                  객체를 검색할 때 쓰이는 메서드들.<br>안에 있는 값이 비어있는지 확인해주는 `isEmpty()`, 객체 저장여부를 체크해주는 `contains()`, 저장되어있는 객체의 갯수를 출력해주는 `size()`가 있다. <br>isEmpty와 contains는 boolean값으로 return해주고, size()는 숫자로 return해준다.                                                   |
| `Iterator()`                    | 전체에 어떤 객체들이 집합안에 있을때, 그것들을 한번씩 반복해서 객체 검색을 해주는 메서드. 가르키는 대상의 원소를 명시해주어야한다.<br>return값은 `Iterator<E>`이며 Iterator의 메서드는 매개변수가 없다.<br>메서드로써는 다음 원소가 있다면 true를 반환하는 `hasNext()`, 해당 객체 타입으로 실제로 가져와주는 `next()`, 리턴은 없지만 객체 제거시 사용하는 `remove()`가 있다. |

<br>

특히 treeset같은 경우, 리턴타입이 전부 객체이다.

<br>

| treeset 메서드  |                                                     설명                                                     |
| --------------- | :----------------------------------------------------------------------------------------------------------: |
| `first()`       |                                              제일 낮은객체 리턴                                              |
| `last()`        |                                             제일 높은 객체 리턴                                              |
| `lower(객체)`   |        주어진 객체보다 바로 아래 객체 리턴 예시로 lower(3)일시 3보다 작은 바로 아래노드를 리턴해준다.        |
| `higher(객체)`  |  lower와는 반대 개념, 주어진 객체보다 바로 위 객체를 리턴한다. ex)higher(6) => 6보다 큰 바로 위의 노드 리턴  |
| `floor(객체)`   | 주어진 객체랑 `동등한 객체`가 있을시 리턴. 없을시 주어진 객체의 바로 `아래의 객체`를 리턴. ex) 반올림의 내림 |
| `ceiling(객체)` |                floor와 같지만, 동등한 객체가 없을시 바로 `위의 객체`를 리턴. ex)반올림의 올림                |
| `pollFirst()`   |                                 제일 낮은 객체를 꺼낸뒤 컬렉션에서 제거한다.                                 |
| `pollLast()`    |                              pollFirst와 비슷하지만, 제일 높은 객체를 꺼내온다.                              |

## map collection

`객체의 형태를 가진 키<key>와 값<value>의 페어형태의 구성을 가진 엔트리 객체`를 저장하는 구조를 가진 컬렉션이다.  
파이썬의 dicionary와 비슷한 구조를 가지고있다.

<br>

선언의 예시  
`map<string,Integer> f = new HashMap<String, Interger>();`

<br>

`키는` 배열의 인덱스처럼 `그 자체로 식별을 하기 때문에` 중복저장 할수는 없으나 `값은 중복인게 있을수 가 있다.`  
기존에 저장된 키와 동일된 값을 저장하면 기존의 데이터를 덮어씌우는 형태로 새롭게 저장이 된다.

<br>

map collection 으로는 `HashMap, HashTable, Link Hashmap, ProperTies, TreeMap` 등이 있다.  
키로 객체를 조작하기에 키를 매개변수로 가진 메서드들이 많다.

<br>

| 관련 메서드              |                                                 설명                                                 |
| ------------------------ | :--------------------------------------------------------------------------------------------------: |
| `put();`                 |           객체 추가 메서드, 주어진 키와 값을 추가할때 사용한다. 저장시 해당값을 리턴한다.            |
| `ContainsKey(객체);`     |            매개변수로 들어온 키 값(객체)이 존재하는지 확인한다. 있다면 true, 없다면 false            |
| `ContainsValue(객체 값)` |                    ContainsKey와 비슷하지만 키 값이 아닌 값(value)에서 확인한다.                     |
| `entrySet()`             | key-value 페어구조로 된 모든 map.entry객체를 담아서 리턴한다. ex) `set<map.entry<k,y>> entry.set();` |
| `get(객체 키)`           |                안에 있는 값(value)를 꺼낼시 사용한다. 해당 키에 해당되는 값이 리턴값.                |
| `size();`                |                                맵 객체 안에있는 키의 갯수를 리턴한다.                                |
| `clear();`               |                    모든 map.entry의 키와 값을 삭제한다. return과 매개변수는 없음.                    |
| `remove(객체 키);`       |       삭제한 해당키의 삭제한 값을 리턴한다. 즉, 리턴후 삭제하는 개념. 파이썬의 pop과 비슷하다.       |
| `Set<k> keySet();`       |  모든 키를 set 객체 - `Set<K>` 에 담아서 리턴한다. set은 집합이기에 출력시 순서가 랜덤으로 나온다.   |
| `Collection<V> Value();` |        리턴타입이 collection으로 저장된 모든값을 collection에 담아 리턴한다. `collection <V>`        |
